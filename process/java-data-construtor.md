## 数据库存储结构

### 行数组：

```
rows: [{
	id: 1,
	preid: 0,
	height: 30
},{
	id:4,
	preid: 1,
	height: 30
}]
```
> 列也是如此

### 单元格结构：

```
cells: [{
	uuid: 6hj32-376dhf-asdjd-dfdf,
	props:{
		...
	}	
}]
```

### 关系表：

`rowid`, `colid`, `uuid`

存储关系的原因：
当一个单元格足够大的时候

```
occpuy:{
	srid: 1,
	scid: 1,
	erid: 3,
	ecid: 3
}
```

这种结构没法找到对应的数据

> * 跟前端数据结构不同点：
> - 数据库存储的关系是平行关系图，进入java还是转化成map关系
> - 删除数据时，后端可以真实的删除，前端不会

### 后记

- 前端后不能共用MAP结构，因为前端不能通过UUID找到对应的数据
前端存储方式为数组
java存储是数据库字段
